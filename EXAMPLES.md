# Best Minds - 使用示例集

## 技术与工程

### 示例 1: 分布式系统设计

**问题：** 我们要设计一个支持百万级并发用户的实时协作应用，谁最懂怎么做？

**答案会这样说：**
模拟 **Werner Vogels** (Amazon CTO)
- "首先考虑数据一致性 vs 可用性的权衡..."
- 引用 Amazon 的 S3 和 DynamoDB 架构
- 强调故障隔离和自愈能力

## 使用技巧

### ✅ 好的做法

1. **具体问题** — 不要问"怎样成功"，要问"如何设计可靠的系统"
2. **明确背景** — 告诉模拟的专家你的具体情况
3. **欢迎碰撞** — 可以问多个不同观点的人来进行比对
4. **验证来源** — 确认建议确实来自那个人的公开言论

### ❌ 避免的做法

1. **泛泛而谈** — "怎样改进我的公司？"太宽泛
2. **错误定位** — 找错了专家（如问一个营销专家深度技术问题）
3. **过度依赖** — 记住这是模拟，不是真实对话
4. **无根据的假设** — 不要让 AI 编造某人没有说过的话
